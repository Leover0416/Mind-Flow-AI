## 1、构建不同的卡片内容：

```javascript
/** 构建快捷卡片提示词（每次刷新随机种子 + 随机方向） */

function buildSuggestionPrompt() {

const seed = `${Date.now()}-${Math.random().toString(36).slice(2, 11)}`;

const shuffled = [...SUGGESTION_THEME_POOL].sort(() => Math.random() - 0.5);

const themes = shuffled.slice(0, 4).join('、');

return `请随机生成 4 个简短的、适合作为 AI 对话开场的「请求句」，用于展示在首页快捷卡片上。用户点击后会把这句话发给助手，助手需要直接给出回答。

【本次随机种子】${seed}（请忽略本行，仅用于区分请求；请据此联想**不重复、有新意**的具体问法）

要求：

1. 必须用「请…」「给我…」「介绍一下…」「推荐…」等会让助手直接给出内容、推荐或介绍的句式，不要用「你想…吗？」「要…吗？」等会让助手反问用户的疑问句。

2. 每句一句话，4 句之间不要重复；尽量从下面四个**不同**方向各写一句（可自由发挥具体问法，不必生硬套词）：${themes}

3. 避免总写「Python 入门、番茄钟、野餐清单、电影片单」等常见套话。

4. 只输出 4 句，每行一个，不要编号、不要引号、不要其他说明。`;

}
```
解读：
#### **1、随机种子（seed）：保证 AI 生成内容不重复：**
- **Date.now()**：获取当前毫秒级时间戳（比如`1719000000000`），确保 “不同时间” 生成的种子绝对不同；
- **Math.random().toString(36).slice(2, 11)**：
    
    - `Math.random()`：生成 0~1 之间的随机小数（比如`0.8a7b9c8d7e123`）；
    - `.toString(36)`：转成 36 进制（包含 0-9+a-z），结果是`0.8a7b9c8d7e123`；
    - `.slice(2, 11)`：截取第 2 到第 11 位，去掉开头的`0.`，得到`8a7b9c8d7e`；
    
- **拼接后示例**：`1719000000000-8a7b9c8d7e`—— 全局唯一，传给 AI 后让 AI 基于这个标识生成 “不重复的内容”（即使同一主题，也会有不同问法）。
**为什么是36进制，因为：**
字符集足够大，能缩短字符串长度
进制数越大，表示同一个数值需要的字符数越少：
- 10 进制（0-9）：只能用数字，表达范围窄，随机字符串会很长；
- 16 进制（0-9+a-f）：6 个字母 + 10 个数字，共 16 个字符；
- 36 进制（0-9+a-z）：26 个小写字母 + 10 个数字，共 36 个字符；
- 62 进制（0-9+a-z+A-Z）：字符更多，但 JS 原生`toString()`不支持（需手动实现）。
也就是：36 进制用更少的字符就能表达同等随机性。

#### 2. 随机打乱主题池 + 截取 4 个主题
```java
const shuffled = [...SUGGESTION_THEME_POOL].sort(() => Math.random() - 0.5);
const themes = shuffled.slice(0, 4).join('、');
```
- **前置说明**：`SUGGESTION_THEME_POOL`是一个 “主题数组”（比如`['旅行攻略', '美食做法', '职场技巧', '学习方法', '健身计划', ...]`），里面存了各种对话主题方向。
- **逐行解析**：
    - 第一行：`[...SUGGESTION_THEME_POOL]`先复制主题池数组（避免打乱原数组），然后`sort(() => Math.random() - 0.5)`是 JS 经典的 “随机打乱数组” 写法（`Math.random() - 0.5`会随机返回 - 0.5~0.5 之间的数，让数组元素随机交换位置）；
    - 这里`...` 是 JavaScript 的扩展运算符（Spread Operator），这里的 `...SUGGESTION_THEME_POOL` 本质是把数组 `SUGGESTION_THEME_POOL` 的所有元素 “展开” 成独立项，再用 `[]` 重新包裹成一个新数组。
    - 这样写的关键：避免修改原数组。因为之后每次还要根据原来的数组内容打乱。而 `[...原数组]` 会创建一个原数组的浅拷贝，排序时只修改这个拷贝后的新数组，原数组完全不受影响。
    - 先搞懂 **JS 的 `Array.sort()` 方法规则**，再拆解这个 “随机排序” 的逻辑，我们一步一步讲：

javascript中`sort()` 数组排序的方法，它的核心是接收一个「比较函数」作为参数，这个比较函数的返回值决定了两个元素的排序位置：

| 比较函数返回值 | 排序结果               |     |
| ------- | ------------------ | --- |
| > 0（正数） | 第一个元素排在第二个元素**后面** |     |
| < 0（负数） | 第一个元素排在第二个元素**前面** |     |
| = 0（零）  | 两个元素位置不变           |     |
`sort()` 的参数是一个**没有显式写 `a` 和 `b` 的箭头函数**，核心目的是「让排序结果随机」，我们拆两层看：

1. 第一步：`Math.random()` 是什么？
`Math.random()` 会返回一个 **0 到 1 之间的随机小数**（比如 0.123、0.89、0.499 等，永远不会等于 1）。
2. 第二步：`Math.random() - 0.5` 会得到什么？
用上面的随机数减 0.5 后，结果的范围变成：**-0.5 到 +0.5 之间的随机数**：
- 有 50% 概率返回**负数**（比如 0.2 - 0.5 = -0.3）；
- 有 50% 概率返回**正数**（比如 0.7 - 0.5 = 0.2）；
- 几乎不可能返回 0（因为 `Math.random()` 是连续随机数，刚好等于 0.5 的概率无限小）。
1. 第三步：为什么能实现 “随机打乱”？
当 `sort()` 用这个函数作为比较规则时，每一次两两元素对比：
- 50% 概率返回负数 → 前一个元素排前面；
- 50% 概率返回正数 → 前一个元素排后面；
这种 “随机正负” 的结果，就会让数组元素的顺序被彻底打乱，达到 “随机排序” 的效果。
    - 第二行：`slice(0, 4)`从打乱后的数组里截取前 4 个主题，`join('、')`用顿号拼接成字符串（示例：`旅行攻略、美食做法、职场技巧、学习方法`）。
- **核心作用**：每次调用函数，都会随机选 4 个不同的主题方向，保证 AI 生成的 4 个请求句覆盖不同领域，不重复。

- **模板关键规则解读**（保证 AI 生成内容符合业务需求）：
    
    - 强制句式：必须用 “请 / 给我 / 介绍一下 / 推荐” 等指令式句式，避免反问（比如要生成 “请推荐 3 个小众旅行目的地”，而非 “你想了解旅行攻略吗？”）；

    - 避免套话：防止 AI 生成千篇一律的常见内容；
    - 输出格式：只返回 4 行文本，无额外格式（方便前端直接渲染到快捷卡片上）。

### 设计思路总结

1. **随机性**：通过 “时间戳 + 随机字符串” 的种子、打乱主题池，保证每次生成的提示词 / AI 返回结果不重复；
2. **规范性**：通过严格的提示词规则，约束 AI 生成的请求句符合 “指令式句式、多主题、无套话、固定格式” 的业务要求；
3. **可维护性**：把主题池抽离成`SUGGESTION_THEME_POOL`，后续想扩展主题只需修改这个数组，无需改核心逻辑。
